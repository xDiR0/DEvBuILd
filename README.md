# DEvBuILd  
## README — Game (Godot)

Экзаменационный проект: 3D с элементами хоррора и голосовым управлением.  
По настроению — что-то между *Phasmophobia* и *Little Hope*, с механикой голоса как ключевым способом взаимодействия с миром.

Команда: 2 человека

---

## О проекте

Игра строится вокруг исследования пространства и взаимодействия с окружением через голос.  
Игрок не просто перемещается по локации, а влияет на события с помощью уровня голосовой активности.

Главная особенность проекта:  
голос — основная игровая механика.

Проект создаётся как прототип на Godot 4.x с упором на:
- стабильную работу,
- понятную архитектуру,
- сохранение прогресса и настроек,
- демонстрацию практических навыков разработки.

---

## MVP

В минимальной версии проекта реализуются 2 комнаты:

### 1. Тестовая / настроечная
Комната для проверки работы микрофона и голосовой системы:
- горячие клавиши,
- индикация громкости,

### 2. Целевая
Игрок попадает сюда после выполнения голосового условия в первой комнате.  
Например:
- активировать механизм голосом.

### 3. Финальная
Комната с завершением короткого сценария и фиксацией результата.

## Игровая петля

Исследуй → Говори / создавай голосовую активность → Мир реагирует → Меняется состояние сцены → Сохраняй прогресс

---

## Технологии

- Движок: Godot 4.x
- Язык: GDScript
- Голос: локальная библиотека распознавания речи (например, Vosk / PocketSphinx) или внешнее API
- Хранение данных: JSON / ConfigFile в user://
- Контроль версий: Git

---

## Архитектура

В проекте используются:
- Signals — для связи между модулями,
- FSM / состояния — для управления логикой сцен и игровых процессов,
- VoiceService — единый интерфейс голосовой системы.

Пример методов:
- start()
- stop()
- set_sensitivity()
- on_transcript(signal)
- on_activity(signal)
